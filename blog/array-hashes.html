Ruby Hashes and Arrays

Posted on January 20, 2016

Arrays

By definition, "Arrays are ordered, integer-indexed collections of any object". In less technical language, an array is a bundle of variables. To visualize this, image that an array is a storage container, and we can place different things inside this container, be it boxes, loose items, big, small and anything in between, as long as it fits in the container, it can be stored. Now as we know, variables are just abstractions for things that can change, so we can visualize them as boxes, and its contents can change.

Let's take an example. We are going to make a container, and we are going to call it "wheels". And we are going to put three items inside it: the number 5 and a sign that says "your the best!". That would look like this: wheels = [5, "your the best!"]

Now that we have a container, we can look at the second part of the definition, "ordered, integer-indexed", which is fancy technical talk for "first come, first server." That is, each item inside the container "wheels" has been given a number as they are stored, which allows us to do some interesting things. To see it in action, we can ask "wheels" how many items it has like this: wheels.length and the result would be => 2, we can change the contents of the first box in the container like so: wheels[0]="Hive Five", which will result in wheels=["Hive Five","your the best!"].

The index is the slot in the array, so index zero has the value "Hive Five", index two has "your the best!", etc. And its ordered, so the next object that you add to the array will be placed at the end by default, unless you specify its position like this: wheels.insert(1,"big guy,") which will cause the array to be: wheels=["Hive Five","big guy,", "your the best!"]. We can also do fun stuff like concatinate all the items in array like so: wheels.join(" ") and that returns a string "Hive Five big guy, your the best!".

We can also tons of things, like manipulate each element, or only ones that meet a certain criteria through iteration like: wheels.each{|word| word.upcase!}.join(" ") which returns "HIVE FIVE BIG GUY, YOUR THE BEST!". As we can see, we can perform multiple actions, and they will be performed in sequence as they are noted on the command. In your example, each word is converted to all caps in place, so we changed the original array, and then we join all of the items with a space in between to result in string.

Those are just a few things that we can do with arrays; many more can be found by reading the documentation for ruby on the resources belew.

Hashes

Now the definition of a hash by the ruby documentation is as follows: "A Hash is a dictionary-like collection of unique keys and their values." In other words, its an association of one key and one value. Its like a two column matrix, except that the keys are all unique, so there can't be two keys named "John". This can be useful when you want to have a list of items where there is only one connection.

The best way to illustrate this is through an example. Let's say we need to keep track of what city people are located at a specific time. Because you can not be in two places at once, a hash would not allow keys(the person's name) entered into the hash more than once. The hash also doesn't have to be uniform, thus one person can enter their location in long, lat format, while another peorson can input a zip code, or an airport code.

Let's see it in action; we are going to have three people, with their location: location = {Alice:92010, Britney: "LAS", Barack: "Washington, D.C."}. Now if we want to add one to the hash, we simple do the following: location["Barack"] = "White House".

Its important to note, that "Barack" and Barack are two different objects, that is why its allowed. The technical reason behind it is that when we use the location = {Barack:"White House"} style, we are actually doing the following: location[:Barack] = "White House" which is equivalent to location = {:Barack => "White House"}, where as the other form is using a string to associate with another string. Placing the colon infront of the name creates a symbol vs a string. This will affect the hashes performance but only once the hash has a large number of enteries, otherwise the user will not see any difference in performance.

But if we did the following: location = {Alice:92010, Britney: "LAS", Barack: "Washington, D.C.", Barack: "White House"}, there would be an error and the first value would be discarded, and only the last value stored. Hashes are better when we need to search for data, so if we want to know where Alice is, all we have to do is: location[:Alice]. This is much better than remembering a number that corresponds to an index in an array. And if your key is not found in the hash, it will return a nil. Now there are tons of things that we can do with hashes similar to the ones we did for arrays, and you can look at those in the documentation pages.
